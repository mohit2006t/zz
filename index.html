Single Type Accordion

Uses .accordion-single class. Only one item can be open at a time.

How does single-type behavior work?

When you open an item in a single-type accordion, all other items automatically close. This creates a clean, focused interface where users see one piece of content at a time.

This item starts open (using class)

This item uses the .accordion-item-open class to be initially opened. The JavaScript handles this during initialization.

Multiple Type Accordion

Uses .accordion-multiple class. Multiple items can be open simultaneously.

Can multiple items be open?

Yes! In multiple-type accordions, you can have as many items open as you want. This is perfect for FAQ sections or when users need to compare different pieces of information.

Default Behavior

No specific type class. Falls back to multiple behavior.

What's the default behavior?

When no specific type class is provided, the accordion defaults to multiple behavior. This leverages the native HTML <details> element behavior.

This item starts closed (explicit class)

This item uses the .accordion-item-closed class to explicitly start in a closed state, even if the HTML has the open attribute.
